Overview

Entity Component System

The engine handles four types of entities. Statics which are used for world building and cannot move. Mobs which are mobile entities able to move, have AI and interact with the world. Items which are able to be picked up and equipped by mobs. Finally vehicles which can be interacted with and driven by mobs.

Each entity is defined in an external JSON file with a name, dimensions, and a customizable list of components. The components available range from base entity stats like attack and defense to physics properties to AI behaviors. An entity component system reduces hard-coding in the engine and organizes things that need to be easily changed to external scripts and JSON files.
